## 2.1 基本类型与数值语义

### 原章节

- §2 基本数据类型
- §15.2 throw（`divide` 示例）

### 修改动作

- **替换**

### 原规则

- `Float -> float32`
- 但 `divide(a Float, b Float)` 编译示例输出 `float64`

### 新规则（最终）

- `Float` 统一映射到 `float64`（默认浮点类型）
- 新增显式 `Float32` 映射到 `float32`
- `Float64` 保留为 `float64` 的显式别名（可选）

### 新语法示例

```gos
let a Float = 1.2       // 默认高精度浮点
let b Float32 = 1.2
```

### 迁移建议

- 原来所有 `Float` 不需要改代码，只需改规范映射表
- 若确实要 32 位浮点，显式写 `Float32`

## 2.2 多返回值边界（语言层 vs Go 互操作）

### 原章节

- §15.4 Go 互操作
- §19.3 Go 多返回值解构
- 附录 A 决策 8

### 修改动作

- **替换**

### 原规则

- 语言声明“不暴露多返回值”
- 但示例有 `let (n, err) = fmt.Println("hello")`

### 新规则（最终）

- GoScript **自定义函数**：不允许声明多返回值
- Go **外部函数**：允许解构其非 error 多返回值
- Go 的 `(T, error)` / `(T1, T2, error)`：`error` 被自动映射为可失败调用，必须用 `try`/`catch`/`try!`

### 新语法示例

```gos
// Go: func Lookup(k string) (int, bool)
let (v, ok) = lookup("a")

// Go: func ReadX() (int, bool, error)
let (n, ok) = try readX()
```

### 必须删除的原示例

```gos
let (n, err) = fmt.Println("hello")   // 删除
```

### 迁移建议

- 所有 Go `(…, error)` 的调用统一改为 `try` / `catch` / `try!`

## 2.3 联合类型的可实现重定义

### 原章节

- §10.2 联合类型
- §12.4 泛型约束
- §24 不支持的特性（关联值枚举）

### 修改动作

- **替换 + 新增**

### 原规则

- `type StringOrNumber = String | Int | Float`
- 并声称可编译为 Go marker interface

### 新规则（最终）

将“联合类型”拆成两个可实现层：

1. **约束联合（constraint union）**  
   仅用于泛型约束，直接映射 Go type set。
2. **值联合（value union）**  
   仅允许由命名类型（class/enum/newtype）构成，编译为接口 + 变体类型。
3. **原始类型值联合（如 `String | Int`）**  
   v0.2 暂不支持作为值类型定义；仅允许在 `constraint` 中出现。

### 新语法示例

```gos
// 约束联合（允许）
constraint Numeric = Int | Float | Float32
fun sum[T Numeric](xs T[]) -> T { ... }

// 值联合（允许，但成员必须是命名类型）
class Ok(value Int)
class Err(message String)
union Result = Ok | Err
```

### 迁移建议

- `type StringOrNumber = String | Int` 改为：
  - 约束用途：`constraint StringOrNumber = String | Int`
  - 值用途：引入包装类型再 `union`

## 2.4 指针与 Go 互操作

### 原章节

- §24 不支持的特性（指针）
- §18.5 Go 标准库调用
- §15.4 Go 互操作

### 修改动作

- **替换**

### 原规则

- 不支持指针
- 但宣称可直接调用全部 Go/三方库

### 新规则（最终）

- GoScript 常规业务代码继续“默认不需要指针”
- **互操作子集**新增最小指针语法：
  - 指针类型：`*T`
  - 取址：`&x`
  - 解引用：`*p`
- 该能力仅为 Go 互操作保证可达性，不鼓励滥用

### 新语法示例

```gos
let x Int = 1
let p *Int = &x
*p = 2
```

### 迁移建议

- 规范 §24 改为“默认不需要手写指针；互操作场景支持最小指针语法”

## 2.5 泛型语法消歧

### 原章节

- §12.1 泛型函数
- 附录 A 决策 3

### 修改动作

- **替换**

### 原规则

- `<>` 并依赖“标识符后紧跟 `<`（无空格）”消歧

### 新规则（最终）

- 统一采用 `[]` 泛型参数语法（与 Go 泛型一致）
- 彻底移除“无空格消歧”规则

### 新语法示例

```gos
fun map[T, U](items T[], f (T) -> U) -> U[] { ... }
let doubled = map[Int, Int](items) { it * 2 }

class Box[T](value T) {
    fun get() -> T = value
}

type Stack[T] = {
    items T[]
}
```

### 迁移建议

- 机械替换：`<...>` -> `[...]`（泛型声明与调用位置）

## 2.6 `!` 运算符语义拆分

### 原章节

- §14.4 非空断言 `!`
- §15.3 `!` 强制执行

### 修改动作

- **替换**

### 原规则

- `name!`（非空断言）
- `divide(...)!`（强制执行可失败调用）

### 新规则（最终）

- 保留 `expr!` 仅用于**非空断言**
- 将“强制执行可失败调用”改为前缀关键字 **`try! expr`**

### 新语法示例

```gos
let len = name!.length      // 非空断言
let x = try! divide(10, 2)  // 失败即 panic
```

### 迁移建议

- 所有 `call()!`（错误强制执行）改写为 `try! call()`

## 2.7 Lambda 与代码块消歧闭合

### 原章节

- §7.1, §7.2, §7.3
- 附录 C（`{}` 使用总结）

### 修改动作

- **替换 + 新增**

### 原规则

- 同时支持 `{ x -> ... }` 和 `{ it ... }`
- 附录 C 又写“Lambda 依赖 `->` 消歧”

### 新规则（最终）

1. 显式参数 Lambda：`{ x Int -> expr }`
2. 隐式参数 `it` Lambda 仅允许在两种场景：
   - 尾随 lambda
   - 目标类型已知为函数类型（可由类型推断确认）
3. 其他表达式位置若使用无 `->` 的 `{ ... }`，视为语法错误（避免与代码块冲突）
4. 新增 §7.5：`return` 仅局部返回当前 lambda，不做非局部返回

### 新语法示例

```gos
val f (Int) -> Int = { x Int -> x * 2 }   // 允许
items.map { it * 2 }                      // 允许（尾随 lambda）
```

### 迁移建议

- 赋值、参数传递等非尾随位置，尽量使用显式 `->` 形式

## 2.8 `select` 语法修正

### 原章节

- §17.3 select

### 修改动作

- **替换**

### 原规则问题

- 接收分支写法导致“进入分支后又再次接收”，语义不正确

### 新规则（最终）

- `select` 分支使用显式 `case`
- 接收分支支持绑定变量
- 默认分支使用 `default`

### 新语法示例

```gos
select {
    case v <- ch1 -> { process(v) }
    case ch2 <- value -> { print("sent") }
    default -> { print("no communication") }
}
```

### 迁移建议

- 原 `<-ch -> { val v = <-ch ... }` 统一改成 `case v <- ch -> ...`

## 2.9 `class` 与 `type` 的语义分工

### 原章节

- §8 类
- §10.1 结构类型
- §10.5 Distinct Type

### 修改动作

- **替换**

### 原规则问题

- `class` 和 `type = {}` 都像结构体，边界不清
- `type X = Y` 又被定义为“始终 distinct”，与常见认知冲突

### 新规则（最终）

引入明确分工：

1. `class`：命名、可方法、默认引用语义（编译为 `*T` 传递）
2. `type X = Y`：类型别名（alias），不创建新运行时类型
3. `newtype X = Y`：创建 distinct 新类型（替代原 §10.5）
4. `record`（或保留 `type X = { ... }`）作为值语义数据载体，禁止继承

### 新语法示例

```gos
type UserIdRaw = Int64        // alias
newtype UserID = Int64        // distinct

type Point = {
    x Float
    y Float
}
```

### 迁移建议

- 原 §10.5 中的 `type UserID = Int64` 改为 `newtype UserID = Int64`

## 2.10 可见性修正（`private` 语义）

### 原章节

- §8.7 可见性

### 修改动作

- **替换**

### 原规则问题

- 文档中 `private` 实际表达“包内可见”，命名与直觉冲突

### 新规则（最终）

- `public`（默认）：包外可见
- `internal`：包内可见
- `private`：仅类型内部可见（由 GoScript 编译器静态检查保证）

### 新语法示例

```gos
class Person(
    public name String,
    internal age Int,
    private token String,
)
```

### 迁移建议

- 原 `private`（包内语义）批量替换为 `internal`

## 2.11 for-in 双变量顺序

### 原章节

- §5.2 for 循环

### 修改动作

- **替换**

### 原规则

- 数组：`value, index`
- Map：`value, key`

### 新规则（最终）

- 单变量：仍为 value
- 双变量：
  - 数组/切片：`index, value`
  - Map：`key, value`

### 新语法示例

```gos
for item in items { }
for index, item in items { }
for key, value in myMap { }
```

### 迁移建议

- 双变量循环统一交换顺序

## 2.12 `catch` 结合性与优先级

### 原章节

- §15.3 调用可失败函数

### 修改动作

- **新增**

### 新规则（最终）

- `catch` 是后缀运算符，仅绑定左侧最近的“调用表达式”
- 复杂表达式必须加括号

### 新语法示例

```gos
let x = 10 + (divide(1, 0) catch { 0 }) * 2
```

### 原章节

- §21.1 作用域规则

### 修改动作

### 原规则

- 名称冲突时 receiver 字段优先

### 新规则（最终）

## 2.14 Map 索引语义唯一化

### 原章节

- §13.2 Map
- §19.3 解构

### 修改动作

- **替换 + 删除**

### 新规则（最终）

- `m[key]` 始终返回 `V?`
- 不允许对 `m[key]` 使用 `(value, ok)` 解构
- `ok` 风格仅保留给 Go 外部函数真实多返回值

### 新语法示例

```gos
let v Int? = m["key"]
let v2 = m["key"] ?? 0
```

## 3. 语法层新增“硬规范”（建议新增章节：附录 D）

为消除“看起来能写、实现时冲突”的问题，新增正式语法约束。

## 3.1 表达式优先级（从高到低）

1. 后缀：`.` `[]` `()` `?.` `!`(非空断言) `catch`
2. 前缀：`!`(逻辑非) `-` `try` `try!`
3. 乘除模：`* / %`
4. 加减：`+ -`
5. 位移：`<< >>`
6. 比较：`< <= > >= is !is in`
7. 相等：`== !=`
8. 位运算：`& ^ |`
9. 逻辑与：`&&`
10. 逻辑或：`||`
11. 空值合并：`??`
12. 赋值：`= += -= *= /= %=`

## 3.2 Lambda 消歧规则（建议写成文法注释）

- `LambdaExpr := '{' Params '->' Body '}'`
- `ImplicitItLambda := '{' Expr '}'` 仅在尾随 lambda 或期望函数类型处可用
- 其他 `{ Expr }` 视为代码块，不作为值表达式

## 6. v0.2 最终版规则清单（落地口径）

1. 保留：`let/val`、空值安全、`when` 表达式、`catch/try`、扩展方法（本包限制）、`#{}` Map 字面量  
2. 修改：泛型括号、`!` 语义、联合类型定义、`select` 语法、可见性命名、for-in 双变量顺序  
3. 删除：Map 的 `(value, ok)` 解构、`fmt.Println` 拿 `err` 的示例  
4. 新增：`newtype`、`constraint`、`try!`、最小指针互操作子集、表达式优先级表

## 7. 建议的实施顺序（工程落地）

1. 先改规范文档（本提案）
2. 再改 Langium 语法：先 P0 再 P1
3. 再改类型检查器：错误系统、union/constraint、可见性规则
4. 最后改代码生成器：Go 互操作、`try/try!`、`select case`、`newtype`

这套顺序可以保证你每一步都能“可解析 -> 可检查 -> 可生成”，避免重写。
